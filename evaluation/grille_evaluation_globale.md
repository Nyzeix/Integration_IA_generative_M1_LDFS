# Grille d'évaluation globale du cours

## Répartition des notes

| Composante | Poids | Détail |
|-----------|-------|--------|
| Rooms 1 à 6 (pratiques) | 40 % | 20 points par Room, moyenne des 6 meilleures |
| Room 7 (projets guidés) | 20 % | 20 points, 3 projets évalués ensemble |
| Room 8 (projet final) | 40 % | 40 points, évaluation individuelle |

## Critères transversaux

Ces critères s'appliquent à toutes les évaluations :

### Fonctionnel (40 % de chaque note)
- Le code s'exécute sans erreur
- Les résultats sont corrects et conformes aux attendus
- Les dépendances sont documentées

### Clarté (30 % de chaque note)
- Le code est lisible et commenté
- Les observations sont rédigées en phrases complètes
- Les fichiers sont correctement nommés et organisés

### Esprit critique (30 % de chaque note)
- Les limites sont identifiées et documentées
- Les observations reflètent une analyse personnelle
- Les choix techniques sont justifiés

## Pénalités

| Infraction | Pénalité |
|-----------|----------|
| Code qui ne s'exécute pas | -50 % sur le critère fonctionnel |
| Fichier manquant dans le livrable | -2 points par fichier |
| Plagiat de code sans attribution | Note 0 pour la Room concernée |
| Résultats manifestement inventés (non issus d'une exécution réelle) | Note 0 pour l'exercice |
